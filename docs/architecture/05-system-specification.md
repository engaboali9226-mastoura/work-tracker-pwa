# System Specification

**Document Purpose**

This document defines the current implemented system from a specification perspective.

Unlike the architecture document, this specification describes every business object, service, repository and use case that currently exists inside the system.

- Contractor

# 8. Business Rules

Current business rules.

### Work Day

- Every Work Day has a unique key.
- Work Day provides execution context.

---

### Task

- Every Task belongs to a Work Day.
- Category must exist.
- SubCategory must exist.
- SubCategory must belong to Category.
- Task may inherit business context.

---

### Production

- Every Production Record belongs to one Task.

---

### Events

- Successful business operations generate business events.

# Specification Summary

The current implementation provides a modular business-oriented architecture centered around Work Days, Tasks and Production Records.

Business logic is isolated from infrastructure.

Persistence is abstracted.

Application composition is centralized.

The architecture provides a stable foundation for future business growth.
